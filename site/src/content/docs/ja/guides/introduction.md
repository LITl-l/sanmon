---
title: 概要
description: sanmonとは何か、なぜ存在するのか
---

**sanmon**（三門）は、AIエージェントのアクションに対する三門形式検証スタックです。確率的ガードレールではなく、LLM生成アクションに対する数学的に証明された安全性保証を提供します。

## 課題

AIエージェント（ブラウザ自動化、API呼び出し、DB操作、IaCツール）は危険なアクションを生成する可能性があります。現在のガードレールは確率的分類器やパターンマッチングに依存しており、サイレントに失敗し、形式的な保証を提供しません。

## アプローチ

LLMの内部構造について推論する代わりに（不可能）、sanmonはCUE（単一の真実の源）から導出される3つの門を通じて**LLMの出力**を制約し検証します：

```
┌─────────────────────────────────────────────────┐
│  第一門: 制約付きデコーディング (構造的保証)        │
│  CUEから導出されたJSON Schema。                    │
│  生成時に100%の構造的正確性。                       │
├─────────────────────────────────────────────────┤
│  第二門: CUEバリデータ (意味的検証)                 │
│  ビジネスルール、安全性ポリシー、ホワイトリスト。    │
│  違反を検出 → LLMに再プロンプト。                   │
├─────────────────────────────────────────────────┤
│  第三門: Lean証明器 (健全性証明)                    │
│  ポリシーシステムの一貫性、完全性、                  │
│  合成安全性を証明。CIでオフライン実行。              │
└─────────────────────────────────────────────────┘
```

CUEが単一の真実の源です：構造とポリシーを一度で定義し、JSON Schemaは自動的に導出されます。制約付きデコーディングは確率的な世界（LLM）と決定的な世界（形式検証）を橋渡しします。

## 技術スタック

| コンポーネント | 技術 | 選定理由 |
|---|---|---|
| スキーマ + ポリシー | CUE | 構造と制約の単一の真実の源 |
| ランタイム | Go | ネイティブCUEサポート、低レイテンシ、ライブラリファースト設計 |
| 証明 | Lean 4 | モダンな定理証明器、活発なエコシステム、AI親和性 |
| API | gRPC（+ オプションREST） | フレームワーク非依存の統合 |
| CI | GitHub Actions + Nix | 再現可能な自動検証 |

## 差別化

| プロジェクト | アプローチ | sanmonとの違い |
|---|---|---|
| Invariant Labs (Snyk) | Python DSLルールベース | 形式証明なし、ランタイムチェックのみ |
| AWS Bedrock Automated Reasoning | 事実内容の形式論理 | コンテンツ検証、アクション制約ではない |
| Guardrails AI | バリデータコレクション | 確率的検出、数学的保証なし |
| AWS Cedar + Lean | 認可ポリシー検証 | 静的ポリシー、AIエージェントランタイム制約ではない |
| スマートコントラクト検証 | Coq/Lean/Isabelle | コード自体を検証、AI生成アクションではない |

sanmonはCUE定義 + 制約付きデコーディング + ランタイム検証 + メタレベル形式証明（Lean）を組み合わせた唯一のプロジェクトです。
